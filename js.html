Writing JS

JavaScript should be added at the end of the body tag because JS generally will affect HTML elements and a browser loads sequentially. Therefore, if at the top the JS will try to affect elements that haven’t even been rendered yet Additionally, it also affects the load time of the website making it load slower You can add javascript in 3 ways.
Inline
“script”
“script:src”
Although not necessary in most cases using a semicolon ( ; ) at the end of your lines is recommended as there are rare cases where it is necessary to. Addiontally a comment in js is written by using two slashes.

Variables

There are three keywords to declare a variable:
var -
With “var” the value within can be changed var is an older way of creating variables used in earlier versions of JS var has a scope that functions differently that sometimes allows variables inside of a function to be accessed outside of the function. Realistically nowadays there is no good reason to use var It is better to just use “var” and “const”
let -
With “let” the value can also be changed later on
const -
Const is immutable which means that it cannot be changed

Object

Objects are very widely used in programming and as well in Javascript. An object is just like a variable and is defined using “let” or “const” however, it has information stored with curly brackets.
As you can see at the end of the page, an object is written with “name:value”/”key:value” pairs. This allows you to store much more information within a single variable as compared with simple strings and integers.

Arrays

Arrays are similar to objects to an extent and are just another type of variable as a string or integer is. A JS array functions very similar to that of a python list and has a numerical index on its entries, something an object doesn’t have Here is an example of an array:
let fruits = ['Apple', 'Banana']
As seen above an array is declared the same way as any other variable and it has a square bracket around it instead of curly brackets.
